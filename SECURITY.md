## セキュリティ脆弱性の報告

医療情報を扱う拡張機能という性質上、セキュリティは最優先事項です。

### 報告方法

セキュリティ脆弱性を発見した場合は、**公開のIssueで報告しないでください**。

以下の手順で報告してください：

1. **GitHub Security Advisory** を使用して報告
   - リポジトリの「Security」タブから「Report a vulnerability」を選択
   - 詳細な情報を記入して送信

2. または、メールで報告（準備中）

### 報告内容

以下の情報を含めてください：

- 脆弱性の種類と影響範囲
- 再現手順
- 使用環境（OS、ブラウザバージョン等）
- 可能であれば、修正案やパッチ

### 対応プロセス

1. **受領確認**: 24時間以内に受領を確認
2. **初期評価**: 48時間以内に深刻度を評価
3. **修正開発**: 深刻度に応じて優先度を決定
   - Critical: 24時間以内に緊急パッチ
   - High: 7日以内に修正版リリース
   - Medium: 30日以内に修正版リリース
   - Low: 次回リリースに含める
4. **公開**: 修正版リリース後、適切なタイミングで脆弱性情報を公開

## セキュリティ対策

### データの取り扱い

#### ローカルストレージ
- テンプレートデータは全て Chrome Storage API を使用してローカルに保存
- ユーザーデータは外部サーバーに送信されません（AI機能を除く）

#### AI機能使用時
- Claude APIへのデータ送信は**事前に明示的な同意を取得**
- 送信されるデータの内容を事前に確認可能
- 個人を特定できる情報（氏名、住所等）の送信は推奨しません

### 通信のセキュリティ
- 全ての外部通信はHTTPS経由で暗号化
- APIキーはローカルストレージに保存（将来的に暗号化予定）

### 権限の最小化
- 必要最小限の権限のみを要求
- `<all_urls>` 権限: 任意の電子カルテシステムでの動作に必要

### コードの安全性
- Manifest V3 準拠（最新のセキュリティ基準）
- Content Security Policy (CSP) を厳密に設定
- XSS対策: ユーザー入力は適切にサニタイズ

## プライバシー保護

### データ収集
- **個人を特定できる情報は収集しません**
- テンプレートデータはローカルのみに保存
- 使用統計の収集は行いません（将来、オプトイン方式で検討）

### サードパーティサービス
- **Claude API (Anthropic)**: AI機能使用時のみ、ユーザーが明示的に送信したデータのみ
  - プライバシーポリシー: https://www.anthropic.com/privacy
  - データの保持期間: Anthropic のポリシーに準拠

### データの削除
- 拡張機能をアンインストールすると、全てのローカルデータが削除されます
- 手動でのデータ削除も設定画面から可能

## 推奨される使用方法

### 医療情報の取り扱い

1. **個人情報の匿名化**
   - 患者氏名、住所、電話番号等は入力しない
   - 必要に応じて仮名やIDで代替

2. **AI機能の使用**
   - 生成された記録は必ず医療従事者が確認・修正
   - 重要な医学的判断の根拠として単独で使用しない

3. **バックアップ**
   - 重要な記録は電子カルテシステム側で適切にバックアップ
   - テンプレートデータは定期的にエクスポート（将来実装予定）

### 職場での使用

1. **組織のポリシー確認**
   - 職場のセキュリティポリシーを確認
   - 拡張機能の使用が許可されているか確認

2. **共有端末での使用**
   - 使用後はブラウザをログアウト
   - APIキー等の機密情報は保存しない

## コンプライアンス

### 該当する可能性のある法規制
- 個人情報保護法（日本）
- 医療法における記録保管義務
- 各医療機関の情報セキュリティポリシー

**注意**: この拡張機能は診療業務の補助ツールであり、電子カルテシステムそのものではありません。正式な診療記録は必ず認可された電子カルテシステムに記録してください。
