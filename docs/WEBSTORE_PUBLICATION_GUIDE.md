# Chrome Web Store 公開ガイド

このガイドでは、「Chrome to Medical」拡張機能をChrome Web Storeに公開する手順を説明します。

## 1. 準備

### マニフェストの確認
`manifest.json` が正しいか確認してください。
- **バージョン**: リリースバージョン（例: `1.0.0`）になっているか。
- **アイコン**: アイコン（16, 48, 128px）がマニフェストで指定されており、ファイルが存在するか。
- **権限**: 審査の遅延を防ぐため、必要な権限のみを申請しているか。

### Zipファイルの作成
拡張機能のディレクトリをZIPファイルにまとめる必要があります。
**重要**: `node_modules`、`.git`、`azure-functions` はZIPに含めないでください。必要なのは拡張機能のファイルのみです。

**Zip作成コマンド（ターミナルで実行）:**
```bash
zip -r chrome-to-medical.zip . -x "*.git*" "node_modules/*" "azure-functions/*" ".DS_Store" "docs/*" "*.sh" "*.md"
```

## 2. デベロッパーアカウント

1.  [Chrome Web Store デベロッパー ダッシュボード](https://chrome.google.com/webstore/developer/dashboard) にアクセスします。
2.  公開に使用するGoogleアカウントでログインします。
3.  初回のみ、**登録料 5ドル（約750円）** の支払いが必要です。

## 3. アップロードと掲載情報

1.  **「新しいアイテム」をクリック**: 作成した `chrome-to-medical.zip` をアップロードします。
2.  **ストアの掲載情報**:
    -   **説明**: 拡張機能の機能をわかりやすく記述します。「医療記録のAIサポート」などを記載してください。
    -   **カテゴリ**: 「仕事効率化」または「ワークフロー」。
    -   **言語**: 日本語。
    -   **スクリーンショット**: 少なくとも1枚（1280x800 または 640x400）アップロードします。サイドパネルやAI機能の画面を見せてください。
    -   **プロモーションタイル**: （任意ですが推奨）440x280の画像。
3.  **プライバシー**:
    -   **プライバシーポリシーURL**: ホスティングされたプライバシーポリシーが必要です。GitHub Pagesなどで `PRIVACY.md` を公開してそのURLを使用できます。
    -   **権限の正当性**: なぜその権限（例: `sidePanel`, `storage`）が必要なのか質問されます。明確に説明してください（例: 「AIアシスタント画面を表示するため」）。

## 4. 審査と公開

1.  **公開**: 「審査のために送信」をクリックします。
2.  **審査期間**: 通常 **24〜48時間** かかりますが、新規アカウントや広範な権限を持つ場合はさらに時間がかかることがあります。
3.  **ステータス**: ダッシュボードで状況を確認できます。

## 5. 公開後

-   公開されると、**ストアURL**（例: `https://chrome.google.com/webstore/detail/...`）が発行されます。
-   このURLをユーザーに共有してください。
-   ユーザーは「Chromeに追加」をクリックしてインストールできます。
-   新しいバージョンをアップロードする（`manifest.json`のバージョンを上げて新しいzipをアップロードする）と、ユーザーの拡張機能は自動的に更新されます。
